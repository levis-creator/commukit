# Credit (requested, not required)

Commukit is released under the [Apache License, Version 2.0](LICENSE). The
license already requires you to propagate the [`NOTICE`](NOTICE) file when you
redistribute the software (Apache-2.0 §4(d)).

This file is a separate, softer ask. If you build something on top of commukit,
we would appreciate — but do not require — that you mention the project in a
place your users or downstream adopters can see it. Any of the following is
welcome:

- A line in your public documentation
- A mention in release notes for the version that first shipped with commukit
- An entry on an "About", "Credits", or "Acknowledgements" page
- A link on your project's README when commukit is part of the stack

## Suggested credit line

Copy-paste this wherever fits best:

> Powered by commukit (https://github.com/levis-creator/commukit).

Markdown variant:

```markdown
Powered by [commukit](https://github.com/levis-creator/commukit).
```

## Why we ask

Credit helps other teams discover commukit and helps us understand where the
project is being used. It also makes it easier for downstream adopters to find
each other and share operational experience.

## What you still have to do

Regardless of whether you display the suggested credit line, Apache-2.0 §4(d)
still requires that you propagate the [`NOTICE`](NOTICE) file contents in the
distribution of any derivative work. `NOTICE` is the legal attribution
mechanism; this file is only a friendly request on top.
